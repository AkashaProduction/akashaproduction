01Architecture globale

Le site est une application PHP sans framework, construite sur un modele de pages autonomes avec un noyau partage (includes/). Chaque page PHP est un controleur qui traite le POST, prepare ses variables, puis inclut header/footer.

Principes

  • Pas de base de donnees — tout le stockage est en fichiers JSON dans storage/
  • Pas de framework — routage via .htaccess, fonctions utilitaires dans helpers.php
  • Multilingue — systeme i18n maison avec fichiers lang/{fr,en,es}.json
  • Config immutable + overlay — config.php (versionne) + runtime-config.php (gitignore, secrets)
  • Sessions PHP — flash messages, langue, session admin

Flux de requete

.htaccess rewrite → page.php → bootstrap.php → POST handler → header.php → HTML → footer.php

03Bootstrap & initialisation

Chaque page commence par require __DIR__ . '/includes/bootstrap.php'; qui execute :

  1. Demarrage de session PHP (session_start())
  2. Chargement de config.php → tableau $config global
  3. Merge de runtime-config.php si existant (array_replace_recursive)
  4. Inclusion de helpers.php et i18n.php
  5. Traitement du parametre ?lang=xx : change la langue en session et redirige sans le parametre
runtime-config.php est gitignore et contient les secrets (cle Stripe, mot de passe admin). Il est fusionne par-dessus config.php. En production, il faut le creer manuellement sur le serveur. 
// Exemple de runtime-config.php
<?php
return [
    'admin_email' => 'contact@example.com',
    'admin_password' => 'mot_de_passe_ici',
    'stripe_secret_key' => 'sk_live_...',
];

07Configuration

config.php (versionne)

Tableau PHP retournant la configuration complete du site :

  • site — nom, tagline, contact, adresse, hebergeur
  • navigation — liens du menu principal
  • projects — 9 projets portfolio (titre, URL, domaine, statut, description)
  • catalog — tarifs creation/hebergement/domaine, prix packs, questions devis, domaines parents
  • support — departements, sujets, priorites, statuts
  • subprojects — liste des sous-projets
  • admin_email, admin_aliases, admin_password
Attention : Les descriptions de projets dans config.php utilisent des apostrophes typographiques Unicode (U+2019) pour le francais. Ne jamais les remplacer par des apostrophes ASCII sous peine de casser les chaines PHP.

runtime-config.php (gitignore)

Fusionne par-dessus config.php via array_replace_recursive. Contient :

  • Cle secrete Stripe (stripe_secret_key)
  • Mot de passe admin (admin_password)
  • Email admin de production si different

stripe-config.php

IDs publics Stripe (produits, prix, coupons, montants en centimes). Pas de secret.

domain-pricing.php

60+ extensions avec prix de vente. Formule : ceil(prix_HT_o2switch * 1.20 + 1). Contient aussi la liste des TLD populaires pour le dropdown.

08Pages & fonctionnalites

PHP

index.php — Accueil

Hero avec titre, stats, CTA. Section "Qui sommes-nous" avec 4 cards. Grille portfolio avec screenshots dynamiques via WordPress mshots.

PHP

3 sections : creations (3 niveaux), hebergements (3 options), packs pre-combines. Chaque card linke vers /commander avec prefill.

PHP

Formulaire complet avec mode projet, uploads multiples (pdf, docx, jpg, webp). Stocke dans contacts.json, envoie un email admin.

Configurateur interactif : choix creation + hebergement + sous-domaine + domaine personnalise. Calcul temps reel du total avec promotions packs. Verification RDAP des domaines.

PHP

Construit une session Stripe Checkout avec les line items. Supporte le paiement 3x. Redirige vers Stripe.

Marque la commande comme "paid", stocke le session_id Stripe. Affiche le recapitulatif et les prochaines etapes.

PHP

Acces par nom + email. Affiche les commandes avec badges de statut, historique tickets avec fil de discussion, creation de tickets, reponses.

PHP

Dashboard avec compteurs. Gestion des commandes (changement de statut). Moderation des tickets (statut, priorite, reponse). Reponses envoyees par email au client.

Endpoint AJAX. Rate limit (10/min/session). Interroge les registres RDAP par extension. Retourne JSON : disponibilite + prix.

10Integration Stripe

Architecture

  • stripe-config.php : IDs produits/prix publics, montants en centimes, coupons packs
  • runtime-config.php : stripe_secret_key (sk_live_...)
  • checkout.php : cree la session Checkout cote serveur via cURL

Flux

commander.php POST → orders.json (status: pending-validation) → redirect /checkout?order=ID → Stripe API: create session → redirect stripe.com → success_url: /commande-confirmee

Paiement 3x

Si split_payment est active, le montant total est divise en 3 line items egaux. Stripe gere les echeances.

Packs & coupons

Les combinaisons creation+hebergement eligibles ont des prix pack reduits. Les coupons Stripe correspondants sont definis dans stripe-config.php.

14Conventions & bonnes pratiques

Securite

  • Tout output HTML est echappe via htmlspecialchars($val, ENT_QUOTES, 'UTF-8')
  • Les uploads sont filtres par extension (pdf, docx, jpg, jpeg, webp)
  • Les noms de fichiers uploades sont sanitises
  • Le stockage JSON est protege par .htaccess
  • L'authentification admin utilise hash_equals() pour la comparaison
  • Le RDAP checker a un rate limit par session

Nommage

  • Fonctions : prefixe app_ (ex: app_read_json, app_flash)
  • CSS : BEM-like (.solution-card__title, .btn--primary)
  • Variables CSS : prefixe semantique (--bg, --accent, --dark-text)
  • i18n : hierarchie par page (home.title, commander.summary_total)

Patterns recurrents

// Pattern type d'une page
<?php
require __DIR__ . '/includes/bootstrap.php';

// 1. Traitement POST
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    // valider, stocker, flash, redirect
}

// 2. Preparer les variables
$currentPage = 'nom';
$pageTitle = app_page_title(t('nav.nom'));

// 3. Inclure le template
require __DIR__ . '/includes/header.php';
?>
<!-- HTML avec  pour les textes -->
<?php require __DIR__ . '/includes/footer.php'; ?>

Points d'attention pour la maintenance

  • Le CSS utilise un systeme de ?v=XX dans header.php — toujours incrementer apres une modif CSS
  • Les traductions sont dans 3 fichiers JSON — modifier les 3 en meme temps
  • Les prix sont definis a deux endroits : config.php (catalogue) et stripe-config.php (montants Stripe en centimes)
  • Les apostrophes francaises dans config.php sont en Unicode (U+2019) — ne pas les remplacer
  • Le runtime-config.php n'est pas versionne — a recreer manuellement sur un nouveau serveur
